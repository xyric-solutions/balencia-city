# DEV-07 Forms: Templates & Checklists

Loaded on demand for output templates, quality checklists, and validation criteria.

---

## Commit Message Template

```
<type>(<scope>): <subject>

<2-4 sentence summary of what changed and why>

<bullet points for multiple changes, if needed>

Refs: <story-id or task-id if applicable>
```

## Commit Message Quality Checklist

- [ ] Type is appropriate for the change
- [ ] Scope reflects the affected area (or omitted if cross-cutting)
- [ ] Subject is imperative mood, lowercase, no period, <=50 chars
- [ ] Body explains WHAT and WHY (for non-trivial changes)
- [ ] Body wrapped at 72 characters
- [ ] Footer references stories/tasks if applicable
- [ ] No invented details - only describes actual changes
- [ ] Uses business language over technical jargon
